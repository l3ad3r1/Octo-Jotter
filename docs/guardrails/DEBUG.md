# Debugging

- Reproduce the reported symptom before editing when possible.
- Trace the bad state to its origin; fix the cause rather than masking the symptom.
- Iterate with the narrowest failing test or command.
- After a failed attempt, record the observation and form a meaningfully different hypothesis.
- After two identical tool failures, change the method or ask for missing information.
- Treat unexpected output as evidence; verify the edited code is the code being executed.
- Confirm a fix with the original reproduction and the nearest relevant suite.
