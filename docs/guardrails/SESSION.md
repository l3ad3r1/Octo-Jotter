# Session continuity

Use `docs/STATE.md` only for long tasks, handoffs, pauses, or context resets.

- Keep `Goal`, `Now`, `Next`, `Constraints`, `Decisions`, `Done`, `Open items`, and `Failed attempts` concise.
- Record user limits verbatim.
- Record decisions with their reason and failed attempts with observed output.
- Refresh `Now` and `Next` before a pause or context reset.
- On resume, read state and verify the working tree before continuing.
