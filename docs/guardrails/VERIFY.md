# Verification

- Run the narrowest relevant check after the last edit and quote its result.
- A compile or type check proves compilation only; test changed behavior when practical.
- Do not weaken, skip, delete, or loosen a failing check to obtain a pass.
- Scan long verification output for errors, failures, warnings, and exceptions.
- Inspect the final diff for unrelated or accidental changes.
- For multi-part requests, report each deliverable as verified, unverified, or not done.
- For documentation-only work, validate structure, links, required terms, and forbidden terms.
- Never claim success without fresh evidence; provide the exact command needed when unverified.
